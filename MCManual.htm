What is MapChart ?

MapChart is a computer package that produces charts of genetic linkage and QTL data. These charts are composed of a sequence of vertical bars representing the linkage groups or chromosomes. On these bars the positions of loci are indicated, and next to the bars QTL intervals and QTL graphs can be shown. MapChart reads the linkage information (i.e. the locus and QTL names and their positions) from text files. This information has to be calculated beforehand, usually with genetic mapping software such as JoinMap® and MapQTL®.

The charting needs of geneticists vary widely. Therefore MapChart comes with many options to adapt the charts to different purposes. However, no charting package can cater for all situations. Charts will often have to be adapted manually. Therefore MapChart is designed to produce graphic files which can be imported by and edited with other software.

New features and upgrade issues

For an overview of the new features, enhancements and bug fixes since previous versions of MapChart, see the MCWhatsNew.htm file in the MapChart program directory. MapChart 2.0 was a completely rewritten version with many new features and a new user interface. Version 2.1 added the possibility to show homologies between loci on different linkage groups, Chart titles, the automatic vertical alignment of linkage groups to show that they belong to the same chromosome, and page footers with page number, date and/or file name. The main changes in version 2.2 were the "omit" option for locus names, the possibility to export bitmaps (*.bmp) as well as metafiles, and an extended graph file format that allows files to contain data for multiple linkage groups and the reading of MapQTL 5.0 output files.

Version 2.3 added more flexible zooming of the charts and had a more robust import of MapQTL 5.0 / 6.0 output files (the previous version was limited to only a few population types). QTL names can now be suppressed. Also a different installer was used as the previous one occasionally gave problems under Windows 7 and 8.

The current version (2.32) addresses an issue where on some systems MapChart could not create a window, a problem that sometimes arose or disappeared after a Windows update.

Upgrading from version 1.x

The current version of MapChart will import the *.map / *.set file pairs produced by version 1.x. All the old formatting is still valid, with the exception of the color codes. In version 1.x, the default color code was C0, and only codes C0 - C9 were available. The background color was specified separately and did not correspond to a color code. In contrast, the current version can handle up to 1000 color codes, code C0 specifies the background color and C1 the default color. When importing version 1.x files, MapChart automatically assigns the old background color to C0, and C0 - C9 to the new codes C1 - C10. In this way the charts are produced with the same colors as before. However, when editing these charts the new color codes must be used. Further, in version 1.x color codes could be specified with only the number; in the current version the number must be preceded with a "c" or "C".

In version 1.x, the command-line parameter /d=<default directory> could be specified with or without the "/" in front. The current version requires the "/", because the first parameter without "/" is interpreted as a file to be opened. It may therefore be necessary to change any shortcuts referring to MapChart.

Upgrading from version 2.0

Files of version 2.0 are generally displayed unchanged in version 2.3, with two exceptions:

  • The indicators of highlighted loci (indicators are the lines connecting locus names to their position on the linkage group bar): these indicators are now shown bold and in the color of the locus name. This can be changed by clearing the "Highlighting indicators" checkboxes on the Chart options - Bars page.
  • The locus texts (the texts that can be shown in separate columns next to the locus names) are now shown without the formatting (style, color, size) of individual locus names. In order to apply this formatting also to the locus texts check "Text in locus style " on the Chart options – Loci page.

In some cases small changes in the data are needed:

  • If graphs with the name "const" are present, these should be renamed as "const" is now a keyword
  • Comment characters (";") should be preceded by a space if they don't occur at the start of a line to make sure they are recognized as the start of a comment, to conform to the JoinMap / MapQTL standard.

Upgrading from version 2.1

  • If Graph files contained columns with caption "position" or "group" these must be changed, as these words are recognized keywords in version 2.2.

Upgrading from version 2.2

No changes are needed, all existing projects will still produce identical results.

In general, files produced by newer versions may be read incorrectly or not at all by older versions.

Drawing charts

A map window in MapChart has two pages, which can be selected by clicking their tabs: the Data page, which shows an editor where the names, positions and formatting of the map elements are entered, and the Chart page where the charts are displayed.

The drawing of a chart starts with a linkage map, such as produced by JoinMap, with a sequence of linkage groups each with the loci listed in ascending order of position (Example 1). To each linkage group QTL bars and graphs can be added, and segments of the linkage groups can be highlighted. All this information is entered in the Data page (see Data Format), which is displayed when you choose File – New from the Main menu. Map files can also be imported, by choosing File – Open and selecting file type "Map file" (or "All files", if your map file has an extension other than ".map"); in that case the Chart page with the calculated chart is shown.

Chart Options (Tools menu) is used to specify the general appearance of the charts, such as the fonts used for the different elements, the various colors, the size and layout of the pages etc. Finally, for individual elements exceptions to these general chart option can be specified in the Data page, such as a different font size for specific loci, and extra formatting such as fill style, color codes can be added (see Data Format).

The chart pages are displayed by choosing the Chart tab, or by opening a map or MapChart data file. If no charts can be calculated because of errors in the data, or because the charts cannot be fitted on the page, an error message is shown and the Data page is displayed. After editing the data, select the Chart page to see the charts. Some common errors and solutions are discussed in the Problems section.

Once the charts are drawn, the Chart page opens showing the full page as it would appear on paper, including the margins. Menu commands, buttons and key commands are then available for navigating through the pages, moving around on the current page and zooming in or out (see Chart menu).

The data can be saved and printed by selecting File – Save or File – Print data from the Main menu. Chart pages can printed (Chart – Print), or copied or exported as Enhanced Metafiles (Chart – Copy or Chart – Export). See the Problems section for solutions to some common problems with importing charts into word processing and presentation software.

Files and directories

MapChart datafiles have the extension *.mct. In addition, MapChart can read the *.map / *.set file pairs produced by MapChart version 1.x; on reading these files the color codes (that followed a different scheme in version 1.x) will be updated automatically. If a *.map file without corresponding *.set file is opened (such as map files exported by Joinmap) the Default Chart Options will be used. If a chart contains graphs, for each graph line the specified graph file must exist.

The Open file and Save file dialogs start in the last used working directory. If this directory is not accessible a default working directory will be used. This default directory can be specified with the command-line option /D=<dir>. If no default directory is specified, c:\ will be used.

MapChart can export chart files, each containing a page with one or more linkage group charts, in the same directory as the *.mct file. These chart files have the name of the *.mct file followed by a three-digit page number, and extension ".emf", indicating that they are in "Enhanced Metafile" format, or ".bmp" for Bitmap format. For example, exporting metafiles from a project named apple.mct will produce apple001.emf, apple002.emf and so on. These files can be imported into other applications such as word processors and presentation software. Please note that exported Bitmaps will generally be very large; in the order of 50-60 MB for an A4 page at a resolution of 600 dpi. Almost always Metafiles will be preferable to Bitmaps.

Also individual pages can be copied to the Clipboard (select Chart menu – Copy page or the Copy Page button) and pasted into other applications that can handle enhanced metafiles. It may be necessary to use "Paste special" or "Paste as metafile" instead of "Paste" in the receiving application. See the Problems section for solutions to some common problems with importing charts into word processing and presentation software.

Linkage groups versus Chromosomes

In general, MapChart draws all linkage groups side-by-side as independent units. Linkage group declarations can start with the keywords "Group" or "Chrom"; they have exactly the same effect, and the "Chrom" keyword is only included for compatibility with older JoinMap versions.

However, sometimes it is known that two or more linkage groups represent different parts of the same chromosome. In this case it is customary to draw these linkage groups aligned vertically, one above the other. MapChart offers a possibility to do that, using the Align vertically parameter in the linkage group header. This parameter creates an aligned group of linkage groups, which are treated as a unit. This affects not only the drawing of the maps, but also the drawing of Homologs: Homologs between linkage groups within the same "aligned group" are drawn like homologs within the same linkage group. Also the Combine Maps tool treats these aligned groups as units.

If the Split Map chart option is active, the maps of long linkage groups are divided into parts that are drawn side-by-side. In that case, vertical alignment becomes pointless and the Align vertically parameter is ignored.

Marker distribution analysis

MapChart can help to study the distribution of different marker categories over the map. This can be useful for selecting e.g. AFLP primer combinations that together provide a good map coverage. There are two possibilities to do so:

  • Manually: Give markers of different categories (e.g. different primer combinations) different color codes (which is perhaps most easily done using a spreadsheet), create sufficient colors and set all the color values initially to black (Chart options - Colors&Lines page). By setting selected color codes to a contrasting color the distribution of the corresponding marker categories is clearly shown. This method is shown in Example 5.
  • Automatically: If the marker categories can be recognized in part of the marker names, it is possible to give each marker category a different color using the Homologs function. On the first Chart options - Homologs page, check Show Homologs and set the "Compare loci" options to compare the relevant part of the marker names. Check Hilight loci - Font color and select "One color per homologs set". Also, disable the drawing of lines between homologs on different and on the same linkage groups (second Homologs page).
    If the number of marker categories is larger than the number of defined color codes, the extra color codes are created but initially all given the same color. Use the Chart options - Colors&Lines page to give them each a different color value.

Data format

MapChart reads linkage and QTL data from text files. These files must contain "flat text", such as produced by a text editor. MapChart cannot read text files in word processor formats. The text is displayed in an editor, which can be used to enter, inspect and modify the data.

The text containing the map data specifies a sequence of linkage groups. Each linkage group specification consists of a header line, followed by a loci section and optionally a QTLs, Graphs and/or a Segments section. Blank lines and lines containing only comments (i.e. lines where the first non-blank character is ";") may occur anywhere in the file and are ignored. Comments may also appear behind other information on a line: in that case the ";" must be preceded by blanks.

It is possible to let MapChart ignore loci or whole linkage groups temporarily by "commenting out" the relevant lines. In order to do so, place the ";" character at the start of all relevant lines; by removing this character the information is made available again.
The editor provides two functions to quickly "comment" and "uncomment" a whole selection of lines (choose Edit – Comment lines or Edit – Uncomment lines from the main menu).

Many parameters for controlling different aspects of the chart elements are discussed below. An overview of these codes is available in the file "MapChart formatting codes.rtf" in the MapChart program directory. This file can be read and printed by any word processor.

Contents Index

Linkage group header

The specification of a linkage group starts with the linkage group header. The header line of a linkage group starts with the keyword "group" or "chrom" followed by the linkage group name. The name will be shown with upper and lower case characters as entered. If the name contains blanks it must be enclosed by double quotes. If no name is desired, enter "" (two consecutive double quotes) if any of the other options are used.

After the name the following optional parameters may be supplied, in any order but all on the same line:

  • S=<number> (Start position, <number> in cM): set the Start position of the linkage group bar to a point (in centimorgan) before the first locus or QTL
  • E=<number> (End position, <number> in cM): set the End position of the linkage group bar to a point (in cM) beyond the last locus or QTL
  • V=<number> (Vertical shift, <number> in mm or map units): Normally the tops of all linkage group bars are aligned, using the method chosen in the Chart options – Bars dialog. This parameter allows to shift the bars of individual linkage groups up or down by the number of mm specified. Useful for aligning corresponding loci on different linkage groups. If you wish to specify the vertical shift in map units instead of mm, set the "Vertical shift units" option to "map units (cM)" in Chart options - Bars
  • H=<number> (Horizontal shift, <number> in mm): The default space left of a linkage group chart is set in Chart options - Page layout; for individual linkage groups this space can be made smaller or wider by setting this parameter (in mm). Useful for visually grouping linkage groups, or to have a larger space between linkage groups than between segments of the same linkage group.
  • P: Force a page break before this linkage group. Useful for visually grouping linkage groups.
  • R: Reverse the linkage group vertically, by measuring all locus, QTL and Graph point positions from the linkage group end instead of from the start.
  • M: Mirror the location (right vs. left of linkage group bar) of locus names and QTL bars with respect to the current chart options. Useful for pairwise comparison of maps, e.g. of the paternal and maternal map of a linkage group. Note: Graphs will always appear on the right side.
  • A: Align vertically with the previous linkage group. All consecutive linkage groups with this option and the last preceding linkage group are grouped into an "Aligned group" representing parts of the same chromosome (Example2 - formatting; see also Linkage groups versus Chromosomes). The effect is that the chromosome bar, Locus text columns, QTL interval bars and Graphs of all linkage groups in the Aligned Group are vertically aligned; only the Vertical shift has to be specified for each linkage group, so that they appear under each other without overlapping. The name of the first linkage group is used as the name of the whole Aligned group. Some formatting options are taken from the first linkage group and ignored in the others; these include the Horizontal shift, Page break and Mirror options. An Aligned group is treated as one linkage group when showing Homologs and in Combine Maps.
  • B, I and U: toggle the Bold, Italic and Underlined style of the linkage group name font on or off, with respect to the name font specified in Chart options - Page layout.
  • C<number>: set the Color code of the linkage group. This affects the name, linkage group bar, locus positions or intervals, and graph outline and font, and also specifies the default value for locus names, bar segments, QTLs and graph lines and symbols. The color specified by the Color code is defined in Chart options - Colors&Lines. The default color code is C1, which is by default set to black. Non-defined color codes use the same color as C1. C0 is the background color.
  • F0 … F7: set the Fill style for the linkage group bar. The default value is F0 (empty bars). The meaning of the codes is shown in Help – Fill styles.

Loci section

The names and positions of the loci (or markers) in the linkage group are specified in the Loci section. The loci section starts on the first line after the linkage group header, and continues until the start of the Segments, QTLs or Graphs section or until the next linkage group header. Each locus is defined on one line, which contains the following elements in this order:

  • Locus name: Shown with upper and lower case characters as entered. If the name contains blanks it must be enclosed by double quotes. This is not recommended as it makes for confusing charts.
  • Locus position: The loci must be listed in increasing order of position. Negative positions are allowed (useful for adding new loci in front of an existing linkage group)
  • Formatting information (optional): includes the font style (B, I and U) and color code (C<number>) as described under linkage group header. Specific formatting information for loci further includes:
    • T1, T2 or T3=<text>: extra Text fields. Up to 3 texts can be specified per locus. These are shown in 3 separate columns, if one locus name per line is shown (see Chart Options - Loci; Example6 - auto-QTLs). The <text> must be enclosed in double quotes if it contains spaces, or if it starts with ";".
    • S<number>: the font Size (in points) for the locus name and text.
    • H<number>: the Homolog number of the locus; this number must be positive or negative, but not 0. Two loci sharing the same positive Homolog number are considered homologs, two loci sharing the same negative Homolog number are considered not to be homologs. This overrules any homology based on similarity of locus names. For a further discussion of homologs see the Chart options - Homologs tab.
    • O: Omit locus name and position from chart. "Omitted" loci are completely ignored in drawing the charts, except that their position is indicated as a tick in the linkage group bar. Useful to draw skeleton maps showing the positions of all used markers, but only the names of a few selected ones.

      The font and color information affects only the locus name and optional text, not the position or interval length numbers. The default color is the linkage group color (C1 if not set in the linkage group header).

Segments section

Segments of the linkage group bar can be shown highlighted, with a special color or fill style (Example3 - homolog matching). These segments are specified in the Segments section. The Segments section starts with a line beginning with the keyword "Segments"; the rest of this line is ignored. The Segments section continues until the start of the QTLs or the Graphs section or until the next linkage group header. Each segment is defined on one line, which contains the following elements in this order:

  • Start position: given in cM. If the start position is before the start of the linkage group, then the round tip of the bar is filled, otherwise it remains empty.
  • End position: given in cM. If the end position is beyond the end of the linkage group, then the round tip of the bar is filled, otherwise it remains empty.
  • Formatting information (optional): includes color code (C<number>) and fill style (F0 to F7) as described under linkage group header. The default color code is the linkage group color (C1 if not set in the linkage group header); the default fill style is F1 (solid fill).

The order in which the segments are defined is not relevant. Also the start and end of a segment may be reversed. The fill style of the linkage group (defined in the linkage group header) should be F0 (default, empty bar), and segments should not overlap, otherwise the printed result may differ from the screen display.

Contents Index

QTLs section

Along the linkage group map, bars can be drawn that show the extent of QTL support intervals. These QTL bars are specified in the QTLs section. The QTLs section starts with a line beginning with the keyword "QTLs"; the rest of this line is ignored. The QTLs section continues until the start of the Segments or Graphs section or until the next linkage group header. Each QTL is defined on one line. QTLs can be specified in two ways: as fully specified QTLs or as Auto QTLs.

Fully specified QTLs

  • QTL name: Shown with upper and lower case characters as entered. If the name contains blanks it must be enclosed by double quotes.
  • QTL intervals: For each QTL two intervals are specified: an outer (e.g. 2-LOD) and an inner (e.g. 1-LOD) interval. The positions of these intervals are specified in the following order: Start of outer interval, Start of inner interval, End of inner interval, End of outer interval, in increasing order (Example4 - QTLs and split map).
    If you want to show only the outer or only the inner interval, specify a zero length for the other one: Let a be the start and b the end of the interval you want to show. To show only an outer interval, enter the values a, a, a, b; to show only an inner interval enter a, a, b, b.
  • Formatting information (optional): includes the font style (B, I and U), color code (C<number>) and fill style (F0 … F7) as described under linkage group header. The default color code is the linkage group color (C1 if not set in the linkage group header); the default fill style is F1 (solid fill). Further formatting for QTLs includes:
    • L0 … L5: line style for the outer interval. See Help – Line styles; default value is L1 (solid lines).
    • W: Wide (bold) lines. The width of normal and wide lines is specified on the Chart options - Colors&Lines tab.

Auto QTLs

Auto QTLs are derived from a graph (see next section and Example6 - auto-QTLs) with exactly the same name. An Auto QTL is located around the graph maximum, with the inner and outer intervals extending to the positions where the graph falls below the maximum minus the "inner threshold" and the "outer threshold" respectively.

A line specifying an Auto QTL is composed of the following elements:

  • QTL name: Shown with upper and lower case characters as entered. If the name contains blanks it must be enclosed by double quotes. A graph with exactly the same name must be specified in the Graphs section.
  • The key word "auto", without quotes.
  • Inner and Outer thresholds: to obtain an inner 1-LOD and an outer 2-LOD interval specify 1 and 2 respectively. Thresholds must be equal to or larger than 0 and the outer threshold must be equal to or larger than the inner threshold
  • Formatting information (optional): includes the font style (B, I and U), color code (C<number>) and fill style (F0 … F7) as described under linkage group header. The default color code is the linkage group color (C1 if not set in the linkage group header); the default fill style is F1 (solid fill). Further formatting for QTLs includes:
    • L0 … L5: line style for the outer interval. See Help – Line styles; default value is L1 (solid lines).
    • W: Wide (bold) lines. The width of normal and wide lines is specified on the Chart options - Colors&Lines tab.

The order in which the QTLs are specified is important. If a QTL fits behind the previous QTL (i.e. it starts at a higher map position than the previous QTL ends, and the names do not overlap), then they are placed in the same "QTL band": a vertical column of QTLs. Otherwise a new QTL band is started, one step further from the linkage group bar (Example4 - QTLs and split map).

Graphs section

On the right side of the linkage group map, graphs can be drawn that show how some parameter (e.g. a QTL LOD value) varies along the map (Example6 - auto-QTLs). These graphs are specified in the Graphs section. The Graphs section starts with a line beginning with the keyword "Graphs". The rest of this line is used to specify the Y-axis formatting and can contain the following elements:

  • L=<number>: the Low limit of the Y-axis; ignored if larger than the minimal value of any of the graphs
  • H=<number>: the High (upper) limit of the Y-axis; ignored if smaller than the maximal value of any of the graphs
  • S=<number>: the Step size (tick interval) of the Y-axis
  • D=<number>: the number of decimals shown

The Graphs section continues until the start of the QTLs or Segments section or until the next linkage group header. Each line in the Graphs section specifies either a constant (a line with constant Y value over the length of the linkage group) or a graph.

A graph is defined on one line, which contains the following elements in this order:

  • Graph name: Shown with upper and lower case characters as entered. If the name contains blanks it must be enclosed by double quotes. Graphs without name will not be shown in the legend; in that case the name must be specified as "" (two consecutive double quotes without space between).
  • Graph file name: the file with the coordinates of the points. If the graph file is not in the same directory as the MapChart file a path must be included. File names including spaces must be enclosed by double quotes. The format of the Graph file is outlined below.
  • Formatting information (optional): includes the font style (B, I and U) for the legend entry, and color code (C<number>) as described under linkage group header. The default color code is the linkage group color (C1 if not set in the linkage group header). Further formatting for graphs includes:
    • L0 … L5: line style. See Help – Line styles; default value is L1 (solid lines). Lines can be suppressed by specifying L0.
    • S0 … S9: symbol style. See Help – Symbol styles; default value is S1 (solid squares). Symbols are shown only at the points as specified in the Graph file. With S0, no symbols are shown.
    • G=<linkage group name>: specifies the linkage group to read from the Graph file; by default the name of the current linkage group is used. This is useful if the linkage group names in the Graph file are different from the names in the MapChart file. Note that group names should be exactly identical, upper/lower case relevant.
    • W: Wide (bold) line. The width of normal and wide lines is specified on the Chart options - Colors&Lines tab.
    • T=<column title>: specifies which column in the Graph file to use for the function values; if omitted, the "LOD" column is used by default (see MapQTL data files to see the default in that case). The column name can be entered in upper or lower case; it may not contain spaces.

A constant (e.g. indicating a LOD threshold value) is similarly defined on one line, as follows:

  • The keyword "const"
  • The Y value
  • Formatting information (optional): exactly as for a graph. The T parameter is of course meaningless for a constant.

The keywords ("group", "chrom", "segments", "QTLs", "graphs", "auto", "const") and the formatting information can be entered in upper or lower case. All names, texts etc. are shown with upper and lower case characters as entered.

Examples of MapChart files showing and explaining all these options are installed in the MapChart\Examples directory.

Graph files

Graph files contain the coordinates of all the points specifying graph lines. From version 2.2 the format of graph files has been extended to allow data for multiple linkage groups to be specified in the same file, which should reduce the clutter of files.

A Graph file starts with a line of column titles; subsequent lines contain the values for a series of points (in order of increasing map position, and grouped per linkage group). MapChart uses the value in column "cM", "map" or "position" as the map position of the point, and by default the value in the column "lod" as the function value. If present, the column "group" lists the linkage groups; this is needed in files containing data for multiple linkage groups. If a value (or any text) is present in the last column, a symbol is shown at that point in the graph. Instead of the "lod" column, any other column title may be specified for the function value, using the T=<column title> formatting option of the graph. In this way, one graph file can define multiple graphs, each specified using a different column title. Using the comment character (";"), comments may be added at the end of a line or between lines.

The keywords "cM", "map", "position" and "group", and the column name for reading the Y-values are not case-sensitive. In contrast, the name of the linkage group listed in the "group" column must be exactly identical to the name of the linkage group in the MapChart file. If the group names in the Graph file are different from those in the MapChart file, the G=<groupname> option can be used to specify which group name to read from the graph file.

Graphs may contain points with map positions beyond the linkage group start or end, but only the part between the linkage group start and end will be shown. Points with function values outside the specified Low or High ends of the Y-axis will cause the Y-axis range to be extended. Further, if the file does not contain a "group" column, or if only one group name occurs in this column (even if different from the linkage group where this graph belongs), all the lines of the file are read.

For compatibility with earlier MapChart versions, the line of column titles may be omitted. In that case the first column should contain the map positions, the second column the function values, and if a third number or text is present on a line a symbol will be shown at that point. Such files define a graph for a single linkage group.

MapChart can read the output of MapQTL 3.0 and 4.0, as exported using the "Copy text" command, and the output of MapQTL 5.0 and 6.0 as exported using the "Export to File" command. By default, MapChart will use the "LOD" column for function values in the case of Interval mapping or MQM mapping, and the "K*" column in the case of Kruskal-Wallis analysis. Don’t edit the MapQTL files, e.g. by adding lines, columns or comments, else the format may not be recognized.

Chart options

The "Chart options" item on the Tools menu or toolbar displays the Chart options form, which is divided into several pages: Page size, Title&Footer, Page layout, Colors&Lines, Bars, Loci, QTLs&Graphs and Homologs.

Page size

MapChart produces output in the form of one or more pages. In the region inside the margins a chart title may be shown at the top. Below the chart title, one or more linkage group maps are drawn side by side. The printed output consists of the pages with the specified margins; the produced metafiles (and the page images copied to the Clipboard) only contain the rectangular region containing the charts, excluding the page margins. Use the Page size tab to specify the paper size and orientation, and the page margins.

  • Paper size: Choose from three pre-defined paper sizes, or define a custom size, and select the page orientation.
  • Margins: an error message will appear if the area inside the margins becomes too small. Also, if you intend to print the charts, take into account the unprintable margins of the printer.

Contents Index

Title&Footer page

Optionally, a chart title and a footer can be shown on the pages. On the Chart options - Title&Footer tab, the following options can be set.

  • Chart title: If no title is entered, no space is reserved for the title and the "title bottom margin" (see below). The chart title, if entered, can appear on the first or on all pages, and can optionally be followed by the page number.
  • Title font button: Shows a font dialog. The chart title is shown in the selected font, using the default color (color code 1, black if not changed)
  • The title bottom space is the space separating the chart title from the top of the linkage group names.
  • Footer: the items to appear at the left, center and right position can be selected. If all three are set to "none", no footer will be shown and more space is available for the charts.
  • Footer font button: Shows a font dialog for the footer

Contents Index

Page layout page

In the area between the title and the footer the linkage group maps are shown side by side. Directly below the chart title is a region where the names of all linkage groups appear, and below that is the "Chart area". The Chart area contains the linkage group bars and all surrounding objects such as the loci, QTL bars and graphs.

On the Chart options - Page layout tab, the following options can be set.

  • Names font button: Shows a font dialog for the names of the linkage groups. For individual linkage groups the font style options (Bold, Italic an Underline) can be changed by adding the B, I or U options (see linkage group header)
  • The names bottom space is the space separating the linkage group names from the top of the chart area. In this space no bars or other chart objects will appear.
  • Multiple linkage groups per page: if unchecked, each page will contain only one linkage group chart. If checked, the default horizontal spacing of charts must be specified in mm; this can be changed for individual linkage groups (see linkage group header)
  • Width of linkage groups: The space reserved for the linkage group can be the actual width of the chart, the width of the widest chart, or a specified width. If the second or third options are chosen, and both locus names and QTL bars are set to appear on the right side of the linkage group bars (see below), then the bars are spaced equally on the page.

Contents Index

Colors&Lines page

In MapChart a color palette is used to specify the colors of the different chart elements. On this page the number of colors in the palette is set, and the actual colors used for each color code are defined. Codes C0 to C10 are pre-defined, but these can be deleted (except codes 0 and 1), changed, or extra color codes can be defined. It is possible to define a large number of extra color codes, which initially all get the same color value (see Add range). This may be useful when dealing with many categories of loci (e.g. AFLP markers based on many different primer combinations): assign each category its own color code, but assign the same color value to all these codes. To highlight the loci of one particular category, just change the color value of its color code (Example5 - marker highlighting and footer).

  • Current color: select the color code to change or delete
  • Change current: displays a color dialog to select a new value for the current color code
  • Delete from current: deletes all codes from the current to the end of the list. Codes 0 (background) and 1 (default) cannot be deleted.
  • Add one: Adds a new color code to the end of the list, and displays a color dialog to specify a color for the new code
  • Add range: displays the "Add color range" dialog, to add a range of new color codes to the end of the list. The new color codes all get the same color value, which can be either selected from the already defined color codes, or specified through a color dialog
  • Background transparent: the background can either be transparent (default, useful for placing the map chart over a background picture) or can be set to a particular color. If a transparent background is selected, the on-screen image shows the "paper" in the selected background color. To show white graphics on a transparent background, choose a dark background color; this will not be included in the metafile image but the graphics will be visible on-screen.

Also on this page the width of normal and wide (bold) lines can be set. Note that these will be approximations: depending on the resolution of your printer usually the specified line widths will be equivalent to only few pixels. Most lines in the charts are "normal"; some line can be specified as wide, including indicators of highlighted loci, QTLs, graph lines and connections between homologs.

Default widths are 0.2 and 0.5 mm for normal and wide lines respectively; maximum values are 1 and 2 mm; wide lines must be at least as wide as normal lines. All lines will be at least one pixel wide.

Bars page

  • Height of chart area: the result of Chart options - Page size (paper size, paper orientation, page margins) and Chart options - Page layout (chart title font, title bottom margin, linkage group names font and bottom margin). The height of the chart area is displayed as an aid in deciding the position and size of the linkage group bars.
  • Alignment: By default, the tops of the linkage group bars are aligned at the same height. If the linkage groups start with different map positions it is possible to align the groups with corresponding map positions at the same height.
  • Tops position: either the longest bar is centered vertically in the chart area, or the position of the top of the linkage group bars (relative to the top of the chart area) can be specified. The second option is most useful for drawing maps from different datafiles with the same layout. The tops of all linkage group bars are aligned by default; individual linkage groups can be shifted up or down (see linkage group header).
  • Vertical shift units: the Vertical shift specified in the linkage group header can be expressed in mm or in map units.
  • Bar length: either the length of the longest bar or the scale (in mm per cM) can be specified. The second option is useful for drawing maps from different datafiles to the same scale.
  • Highlight indicators: if a locus name is highlighted in any way (by specifying different font attributes, font size or color), the indicator connecting it to the linkage group bar can be shown bold and/or in the color of the locus name. This works also if the locus is highlighted because it is a homolog of another locus. If these options are checked, the indicator of a cluster of loci is highlighted if any locus in that cluster is highlighted, and it gets the color of the last highlighted locus in the cluster. (The Bold attribute of the indicators may not be visible in the full-page view; it is always visible at the highest zoom level and on the printed page).
  • Indicator size: the horizontal size of the lines connecting the positions and the locus names to the linkage group bar.
  • Bar width (in mm): the width of the linkage group bar.
  • Split map interval: if set to 0 linkage groups won’t be split up; otherwise linkage groups longer than the interval specified here will be divided into segments and drawn as if they were separate linkage groups, with a number added to the original linkage group name. QTL bars and Graphs will also be split up (Example4 - QTLs and split map).
  • Round tips: when checked, the linkage group bars have round tips, otherwise flat tips. Note: if the linkage group is split into segments (see previous option), only the start and end of the whole linkage group will show a round tip when this option is checked.

Contents Index

Loci page

  • Locus font button: shows a font dialog. The style options (Bold, Italic and Underline) can be changed for individual loci by adding the B, I or U options in the locus lines in the Data page.
  • Locus name position: the locus names can be shown on the left or the right side of the linkage group bar; the position information will appear on the opposite side.
  • Locus names per line: in case more than one locus occurs at the same (rounded) position, all the loci of the cluster are printed in one group. Here the number of loci per line in a cluster is specified. The more loci per line, the easier it will be to fit the chart within the available chart height, but the wider the chart will become. If 0 is entered, no locus names are shown.
  • Show locus texts: Optionally a text can be specified for each locus (see Data format - Loci section). These text will only be shown if this checkbox is checked, and only if one locus name per line is shown (see previous option).
    Locus texts can be useful e.g. to indicate the coupling phase of dominant markers, the significance of skewed segregation or of QTL effects etc.
  • Text in locus style: By default all locus texts are shown in the standard locus font, without the special formatting of the locus name. By checking this option, the formatting of each locus name is also applied to its locus texts.
  • Positions: There are five options: positions of loci can be omitted, the absolute positions, interval lengths or ticks at fixed intervals can be shown along the linkage group bar, or a ruler can be shown at the left side of the page.
  • Decimals: the number of decimals (0-5) used for printing positions and for rounding positions to group loci into clusters (see above).

QTLs&Graphs page

QTL options:

  • QTL font: the QTL font name, font size (in points) and font style can be set here. A sample with the current chart options is shown. The font style options (Bold, Italic and Underline) can be changed for individual QTLs by adding the B, I or U options in the QTL lines in the Data page.
  • Suppress names: if checked only the QTL bars are shown, not the QTL names. Note that the QTL font size still has an effect on the spacing of the QTL bars even if the names are not shown.
  • QTL bar width (in mm): the width of the QTL bars.
  • Suppress names: if checked, no names are drawn at the QTL bars.
  • Show dashes at ends: at the ends of the lines representing the outer interval, a terminal cross-dash can be shown.
  • QTL position: the QTL bars can be shown on the left or the right side of the linkage group bar.

Graph options:

  • Y-axis scaling: selects if the Y-axis of all graphs should have the same length (in mm) or the same scale (in mm per scale unit)
  • Y-axis font button: shows a font dialog. The font is used for the numbers along the Y-axis.
  • Round positions of points: the X (cM) coordinates of all points can be rounded, just as the positions of loci and QTLs or they can be drawn exactly as entered. Rounding is useful to exactly align loci and symbols in graphs but may sometimes cause jagged graphs.
  • Legend font button: shows a font dialog. The style options (Bold, Italic and Underline) can be changed for individual graphs by adding the B, I or U options in the graph lines in the Data page.
  • Show graph legend: if checked, a legend is shown after the last linkage group chart. The legend will omit any graphs without a name.

Homologs pages

MapChart can highlight homologies between loci. Loci can be grouped into homologs groups based on similarity of their names, and/or based on their homolog number which can be specified on the Data sheet. Each locus can be part of only one homologs group. Loci in the same group are shown in the same color and connected by lines, if these options are selected.

If the "Match locus names" checkbox (see below) is NOT checked, loci with the same homolog number are grouped together. If the "Match locus names" checkbox is checked, the loci are grouped in the first place based on name similarity. Further, loci that you wish to remove from such a group you can give a NEGATIVE homolog number (if you give several loci the same negative homolog number, they will form a new group). If you want to add extra loci to a group, give the same POSITIVE homolog number to those loci and to one or more of the loci already in the group.

The Combine Maps tool makes it easy to place corresponding linkage groups from different files side by side, so that homologies can be clearly shown.

The options on these pages specify how to show homologs:

  • Show homologs: if unchecked, no homologies are shown and the rest of these pages are not relevant.
  • Compare loci: If the "Match locus names" checkbox is checked, loci are recognized as homologous if the specified part of their names is identical, possibly ignoring upper/lowercase differences. The default setting "Ignore last 0 characters" only recognizes completely identical names as homologs. See the beginning of this paragraph about how homolog numbers influence this grouping.
  • Highlight loci: The names of homologous loci can be highlighted. If the Font style, Font size and/or Font color checkboxes are checked, the specified formatting is used for homologous loci, overruling the corresponding formatting specified on the Data page. If these three checkboxes are left unchecked (default), locus names are not highlighted. If Font color is checked, you can further choose to show all homologs in the same color or to use a different color for each homologs group. In the last case, color codes 2 and up are used for the homologs groups. If there are more groups than color codes, the extra necessary color codes are generated but they will initially all receive a gray color value. You will receive a warning that you need to specify the desired color values for these extra color codes afterwards (see the Chart options - Colors&Lines page).
  • Connections between linkage groups: Connections are the lines connecting two homologs. They can be shown only between adjacent linkage groups or between all linkage groups on the same page.
  • Connections within linkage groups: Connections can also be shown between homologous loci on the same linkage group. These connections have the form of a half ellipse and can be shown on the left or right side of the linkage group bar. "Width of curve" specifies how wide the half ellipses are at their midpoints.
  • Connection style: A connection between homologous loci consists of a line with a "dot" at each end, at the positions of the loci (or clusters of loci) on the linkage groups.
    The size and color of the dots, and the line style can be specified. A dot size of 0 mm hides the dot; line style "no line" hides the line. The lines can be drawn in normal width or bold. Further, the lines can either all have the same color: the color of the dots, or a different color per homologs group, or a different color per pair of linkage groups. In the second case, if a line connects clusters with several homologs groups, the line will be drawn as several parallel lines in the different colors (it may be necessary to choose Bold lines to see this clearly).
    As described above under Highlight loci, in the last two cases color codes 2 and up are used. If there aren’t enough color codes, the extra necessary color codes are generated but they will initially all receive a gray color value. You will need to specify the desired color values for these extra color codes afterwards (see the Chart options - Colors&Lines page).
    If the two homologous loci are on different pages, the dots are drawn but the line is omitted.

Problems and solutions

No map data available: This error message will appear if MapChart cannot find the start of the data. The data should start with the heading for the first linkage group (starting with the word "group" or "chrom"). If you have other text appearing above the first linkage group, delete it or comment it out (place a semicolon at the start of each comment line).

Error in position of locus: MapChart expects a valid number after the marker name, indicating the position; the loci must be listed in order of ascending position.

Error in point … in graph…: Each line in a graph file should give at least two valid numbers (see Graph files). The points must be in ascending order of X (map position) values.

Error messages stating that the charts don’t fit in the available space: Probably the height of the charts is larger than the space between the page margins. This can be solved by increasing the available page height, and by decreasing the chart height. The page height can be increased by choosing a portrait instead of landscape paper orientation and by setting smaller margins (both through the File – Chart page setup menu option). The chart height can be decreased by changing several chart options (Tools - Chart options menu option or toolbar button). Most effective will be choosing a smaller Locus font size, increasing the number of locus names per line (both on the Chart options – Loci page), and splitting linkage groups into segments ("Split map interval" in the Chart options – Bars page). Also choosing a smaller Title, Name or Y-axis font may help.
Finally shifting linkage group bars up or down may cause them to move outside the chart area; in that case their length must be decreased, or the shifting must be adjusted. The Autofit utility can check out many of these combinations.

Contents Index

Changes in graph files don’t appear in the charts: When MapChart needs to display a chart, it first checks if there are changes on the Data page or in the chart options. If not, no new charts are calculated and the previously calculated charts are shown again. Since changing a graph file does not change the Data page, the charts will not be recalculated. In this case select Recalculate from the Tools menu or button bar: this causes the data and graph files to be read again and new charts to be calculated. Also make sure that you have saved the edited graph file before recalculating the charts!

Charts imported into other applications are printed with very thick lines: When MapChart draws the charts, the width of lines is as specified on the Chart options - Colors&Lines tab, but at least one pixel of the output device. If no Windows printer is installed, the screen resolution will be used, which is very coarse, so even one pixel may appear quite thick. Try generating the charts on a computer with a printer, and the charts will be drawn using the printer resolution. The same problem occurs when the computer is running in "Safe Mode": in that case no printer drivers are installed.

Some chart elements appear distorted after importing into MS-Word or MS-PowerPoint: To avoid changes on importing, make sure that the imported image does not have to be resized. Images exported from MapChart (either by saving as *.emf file, or by copying to the clipboard) contain the area within the margins; by a suitable combination of page and margin sizes (Chart options - Page setup) the desired size can be obtained. Also, exporting the *.emf file from MapChart and importing (Insert Picture from file) seems to give better results than copying via the Clipboard.

Contents Index

Texts and names in charts imported into other applications are not drawn at the correct size: When charts are only imported into word processing or presentation software there is no problem. However, when the charts are edited (especially if they are "split into objects" or "ungrouped") they may be converted to the internal format of the importing application. At that moment, fonts may change a little in size, causing texts to become somewhat wider and to overlap if they are positioned close together. For example, this happens in Microsoft PowerPoint when the "Ungroup" command is used. This is usually almost invisible when charts are drawn at the printer resolution, but can be problematic when drawn at screen resolution (on a computer with no printer installed). It also depends on the font and font size used. If the problem arises, try generating the charts on a computer with an installed printer, and/or try changing fonts or font sizes.

Charts copied to MS PowerPoint are shown very small: If charts are copied to MS PowerPoint through the Clipboard (using the Copy Page button or Ctrl-C), they must be pasted using Edit – Paste Special – Enhanced Metafile in MS PowerPoint. Using the Paste button or Ctrl-V results in a very small picture. Another way to import charts into MS PowerPoint is Insert – Picture – From file, after using Chart export in MapChart.

Contents Index

Graph lines in charts imported into other applications cannot be edited: The dashed and dotted lines in graphs are composed of short line segments. After importing a chart into another application and "splitting to objects" or "ungrouping", the graph lines are not one object any more, and cannot be edited as a whole. The solution is to draw with solid lines (line style L1, the default value) in MapChart; solid lines remain one object after imported charts are split. In the importing application the line style can then be changed as desired.

White lines and texts are invisible: If you need white graphics on a transparent background, e.g. to place them on a dark background in another application, choose a dark color for the background (color code C0) in the Chart options - Colors&Lines page dialog, and check "Background transparent". In MapChart you will see the charts against a dark background, but the copied or exported metafiles will have a transparent background.
